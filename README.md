# Приложение для стеганографии (Java, LSB, PNG/BMP)



## 1. Общая информация о проекте

Данный проект представляет собой учебное десктоп-приложение на языке Java для скрытия текстовых сообщений внутри растровых изображений форматов **PNG** и **BMP** и последующего их извлечения.

Программа реализует метод **стеганографии на основе младших значащих битов (LSB)** и предоставляет простой графический интерфейс на базе Swing. Приложение может использоваться в рамках лабораторных, практических и курсовых работ по дисциплинам, связанным с программированием, информационной безопасностью и обработкой изображений.

Проект разработан в учебных целях и ориентирован на демонстрацию принципов стеганографии, а также на закрепление навыков разработки GUI-приложений на Java.



---

## 2. Цель и назначение проекта

**Цель проекта** – разработка и исследование десктопного приложения, реализующего метод стеганографии с использованием LSB для скрытия текстовой информации в изображениях и её надёжного извлечения.

В рамках проекта решаются следующие задачи:

- реализация базового алгоритма LSB-стеганографии для форматов PNG/BMP;
- встраивание текстового сообщения в растровое изображение;
- извлечение скрытого текста из стеганографического изображения;
- обеспечение наглядного графического интерфейса для работы с алгоритмом;
- демонстрация особенностей использования форматов без потерь (PNG/BMP) при стеганографии.



---

## 3. Реализованный алгоритм стеганографии

В проекте реализован следующий алгоритм:

- **LSB (Least Significant Bit) для цветных изображений** – изменение младших значащих битов цветовых каналов RGB.

Основные особенности реализации:

- встраивание информации в младшие биты каналов **R, G, B** каждого пикселя;
- использование формата данных:

  - первые **4 байта** – длина сообщения в байтах (целое число в формате big-endian);
  - далее – байты текста в кодировке **UTF-8**;

- проверка ёмкости изображения (достаточности числа пикселей) перед встраиванием;
- сохранение альфа-канала (прозрачности) без искажений.

Алгоритм рассчитан на:

- работу с одним изображением за раз;
- форматы без потерь (PNG, BMP), чтобы не нарушать битовую структуру данных.



---

## 4. Модель работы приложения

- Исходными данными являются:
  - растровое изображение PNG/BMP;
  - произвольное текстовое сообщение пользователя.

- При встраивании сообщения:
  - текст переводится в массив байтов (UTF-8);
  - длина сообщения записывается в первые 4 байта служебной области;
  - биты длины и текста последовательно размещаются в младших битах каналов R, G и B пикселей;
  - формируется новое изображение, визуально почти не отличимое от исходного.

- При извлечении сообщения:
  - приложение считывает из изображения первые 32 бита и восстанавливает длину сообщения;
  - затем считывает указанное количество бит и формирует массив байтов;
  - байты интерпретируются как строка в UTF-8 и выводятся пользователю.

- Вся работа с изображением реализована через `BufferedImage` и `ImageIO`, а операции на битовом уровне выполняются с использованием побитовых сдвигов и масок.



---

## 5. Пользовательский интерфейс и функциональные возможности

В приложении реализован графический интерфейс на базе **Java Swing**, включающий две основные вкладки:

- **«Встроить сообщение»**:
  - выбор исходного изображения через стандартный диалог `JFileChooser`;
  - ввод текстового сообщения в многострочное поле `JTextArea`;
  - запуск процесса стеганографии и сохранение нового изображения (PNG/BMP) с текстом.

- **«Извлечь сообщение»**:
  - выбор изображения, содержащее скрытое сообщение;
  - запуск процесса декодирования;
  - вывод извлечённого текста в поле только для чтения.

Дополнительно реализовано:

- отображение путей выбранных файлов;
- вывод сообщений об ошибках и результатах через `JOptionPane` (некорректный формат, сообщение не помещается в изображение, ошибки чтения/записи и т.д.);
- автоматическое добавление расширения `.png`, если пользователь не указал его при сохранении.

## 6. Примеры сценариев использования

Приложение может использоваться в следующих учебных и практических сценариях:

- демонстрация принципа стеганографии на лабораторных занятиях;
- скрытый обмен короткими текстовыми сообщениями через изображения;
- закрепление навыков побитовой обработки данных и работы с `BufferedImage` в Java;
- исследование влияния размера изображения на максимальную длину скрываемого текста;
- расширение проекта за счёт добавления шифрования сообщения перед встраиванием или изменения схемы распределения бит по пикселям.

Сценарии допускают варьирование:

- размеров и разрешения изображений;
- длины текстовых сообщений;
- форматов файлов (PNG или BMP).
